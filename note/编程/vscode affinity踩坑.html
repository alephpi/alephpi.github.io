vscode affinity 踩坑

2024-06-21 · 261字 · 1分钟

上回说到 vscode 的 vim 插件巨慢，然后通过配置affinity属性将它置于单独线程中来加速运行。

然后我就灵光一现，正好 pylance 也很慢，不如我也配置一个单独线程供它使用？一切都很顺利，直到我今天重新开了个 notebook，却发现它死活也找不到 ipykernel 了。在网上转了一圈，都说重启、重装扩展就行了，我不仅都试了一遍，还重启了电脑、重装了 vscode，结果屁用没有。jupyter 的报错始终是

Visual Studio Code (1.90.2, undefined, desktop)
Jupyter Extension Version: 2024.5.0.
Python Extension not installed.
Pylance Extension not installed.
Platform: linux (x64).

这不是睁眼说瞎话吗？我 python、pylance 扩展明明在。然后我突然想到前两天刚刚配置的 affinity 属性，该不会是因为把 pylance 关到单独线程，jupyter 插件找不到它了吧，于是删掉配置，结果就好了。

又是被自己蠢笑的一天，可能全网只有我犯了这么蠢的错误。
